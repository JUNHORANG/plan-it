## 공통 UI

### 바텀 시트 - 공통 UI

1. 바텀 시트의 X 아이콘을 누르면 바텀 시트를 닫을 수 있다.
2. 바텀 시트 외부 영역을 클릭하면 바텀 시트를 닫을 수 있다.
3. 바텀 시트는 아래에서 위로 나타나는 애니메이션으로 노출된다.
4. 목록형 바텀 시트의 경우, 사용자가 선택한 항목은 색상 변경 또는 강조 UI로 표시된다.
5. 기본값이 없을 경우 CTA 버튼은 비활성화 상태이며, 사용자가 유효한 항목을 선택하면 활성화된다.
6. 요청이 발생하는 CTA 버튼은 요청 중 스피너 UI를 표시한다.
7. 요청 성공 시 바텀 시트는 닫히고 성공 토스트를 띄운다.
8. 요청 실패 시 실패 토스트를 띄우고 바텀 시트는 닫지 않는다.
9. 높이 조절이 가능한 바텀 시트는 최대 높이로 확장할 수 있다.

---

### 입력창 - 공통 UI

1. 기본 상태에서는 안내 문구를 표시한다.
2. 입력창에 포커스가 들어가면 테두리 색상이 변경되며 강조된다.
3. 입력을 시작하면 안내 문구가 사라지고 사용자가 입력한 값이 표시된다.
4. 입력창에 블러 이벤트가 발생하면 유효성 검사를 실행할 수 있다.
5. 입력창 값이 검증 된 값이라면 입력창 색상이 변경 된다.
6. 유효성 검사에 실패하면 테두리 색상이 error 색상으로 변경되고 하단에 오류 문구를 표시한다.

### CTA 버튼 - 공통 UI

1. 기본값이 없는 상태는 비활성화 상태이다.
2. 필수 조건을 충족하면 활성화 상태로 변경된다.
3. 클릭 시 요청을 보내거나 다음 단계로 이동한다.
4. 요청 중에는 버튼 내부에 스피너 UI를 표시한다.
5. 요청 성공 시 성공 토스트를 띄운다.
6. 요청 실패 시 실패 토스트를 표시한다.

## 회원가입

도메인: /signup

1. 회원가입 방식
    - 스텝퍼 UI를 사용해 회원가입 단계를 표시한다.
    - CTA 버튼은 기본적으로 비활성화 상태이다.
    - 총 3단계로 회원가입을 진행한다.
2. 헤더
    - 앱 바 - 공통 UI를 참조한다.
    - 뒤로 가기 옆에 제목
    - 뒤로 가기 버튼을 클릭하면 뒤로 가기 모달이 나타난다.
    - 모달 - 공통 UI 참조한다.
    - 뒤로 가기 모달의 취소하기, 뒤로 가기 버튼이 있다.
    - 취소 하기 버튼을 누르면 뒤로 가기를 취소 한다.
    - 뒤로 가기 버튼을 누르면 /login 도메인으로 이동 한다.
3. 1단계 - 닉네임 설정
    - 입력창 - 공통 UI를 참조 한다.
    - 안내 문구 : 한글, 영어, 숫자를 포함한 8자 이내로 작성해 주세요.
    - 닉네임을 입력 할 수 있다.
    - 입력창에 입력이 들어가면 CTA 버튼이 활성화 된다.
    - CTA 중복 확인 버튼을 클릭해 검증 요청을 보낸다.
    - 중복 닉네임이 있다면 에러 ui와 에러 문구를 띄운다.
    - 에러 문구 : 중복 된 닉네임이 있습니다.
    - 검증이 성공 한다면 CTA 버튼이 다음 버튼으로 변경 되고 다음 step으로 넘어간다.
4. 2단계 - 이메일 인증
    - 이메일 입력창에 이메일을 작성할 수 있다.
    - 이메일 입력 시 인증 번호 전송 CTA 버튼이 활성화된다.
    - 인증 번호 전송 버튼을 클릭하면 인증 번호 발송 요청을 보낸다.
    - 이메일 인증 번호 입력창에 인증 번호를 작성할 수 있다.
    - 인증 번호 입력창이 노출되면 제한 시간 UI가 표시된다.
    - 제한 시간은 5분이다.
    - 시간 연장하기 버튼을 클릭할 수 있다.
    - 시간 연장하기 버튼을 클릭하면 인증 제한 시간이 초기화된다.
    - 인증 번호를 입력하면 인증 번호 확인 CTA 버튼이 활성화된다.
    - 인증 번호 확인 버튼을 클릭하면 인증 번호 검증 요청을 보낸다.
    - 이메일 인증이 완료되면 다음 CTA 버튼이 활성화된다.
    - 다음 CTA 버튼을 클릭하면 비밀번호 설정 단계로 이동한다.
5. 3단계 - 비밀번호 설정
    - 비밀번호 입력창에 비밀번호를 입력할 수 있다.
    - 비밀번호 확인 입력창에 비밀번호를 다시 입력할 수 있다.
    - 각 입력창의 눈 아이콘을 클릭하면 암호화된 값을 텍스트로 볼 수 있다.
    - 눈 아이콘을 다시 클릭하면 입력값을 암호화한다.
    - 비밀번호 조건과 비밀번호 검증 조건을 충족하면 다음 CTA 버튼이 활성화된다.
    - 다음 CTA 버튼을 클릭하면 사용자 정보 입력 단계로 이동한다.
6. 회원가입 완료
    - 완료 이미지가 애니메이션과 함께 나타난다.
    - 로그인 하기 CTA 버튼을 클릭할 수 있다.
    - 로그인 하기 CTA 버튼을 클릭하면 로그인 처리 후 /index 도메인으로 이동한다.

## 홈

도메인: /user/plans

- 스켈레톤 - 공통 UI 참조
1. 헤더
    - 헤더 - 공통 UI 참조
2. 다음 일정 정보
    - 사용자는 오늘의 요일과 날짜(월, 일)을 읽을 수 있다
    - 사용자는 다음 일정을 읽을 수 있다.(스켈레톤 필요)
3. 금주의 날짜 선택
    - 사용자는 이번 주의 날짜를 확인할 수 있다.
    - 사용자는 원하는 날짜를 클릭할 수 있다.
    - 날짜를 클릭하면 섹션-3의 내용이 해당 날짜의 데이터에 맞게 변경 된다.
4. 일정
    - 3. 금주의 날짜 선택에서 선택한 날짜의 일정 개수(스켈레톤 필요)를 알 수 있다.
    - 일정 추가 버튼을 클릭해  /user/plans/add 도메인으로 이동할 수 있다.
    - 일정이 없다면 안내 문구를 사용자에게 제공한다.
    - 안내 문구 : 일정을 추가해 주세요!
    - 일정의 목록을 읽을 수 있다.(스켈레톤 필요)
    - 일정의 목록이 시간의 오름 차순으로 나열 돼 있다.
    - 일정의 체크 박스를 선택하면 일정이 비활성화 된다.
    - 비활성화 된 체크 박스를 다시 선택해 활성화 시킬 수 있다.
    - 사용자는 일정의 시간과 제목을 읽을 수 있다.
    - 사용자는 일정의 더보기 버튼을 눌러 바텀 시트를 꺼낼 수 있다.
    - 바텀 시트 - 공통 UI 참조
    - 바텀 시트 : 고정, 수정, 삭제
    - 바텀 시트 기본값 : 고정
    - 바텀 시트  고정 : 고정을 선택하고 CTA 버튼을 누르면 해당 일정을 목록의 상단에 고정 한다.
    - 바텀 시트 수정 : 수정을 선택하고 CTA 버튼을 누르면  /user/plans/edit 도메인으로 이동한다.
    - 바텀 시트 삭제 : 삭제를 선택하고 CTA 버튼을 누르면 일정 목록에서 해당 일정을 삭제 한다.
    - 모든 일정을 비활성화 상태로 만들면 하단에 일정 완료 CTA 버튼이 나오며 도메인  /user/plans/success 으로 이동한다.
    - 로그인하고 첫 index 라우트로 진입하는 사용자에게 온보딩 UI를 보여준다.
    - 온보딩 step-1
        - 건너뛰기 버튼을 눌러 온보딩 UI를 끌 수 있다.
        - 페이지 네이션을 보여준다.
        - 안내 문구를 보여준다.
        - 안내 문구 : 일정을 채우고 완료하면서, 매일 건강한 루틴을 지속해 보세요!
        - 꺽새 아이콘을 눌러 다음 온보딩 UI로 넘어가거나 전 온보딩 UI로 넘어갈 수 있다.
    - 온보딩 step-2
        - 건너뛰기 버튼을 눌러 온보딩 UI를 끌 수 있다.
        - 페이지 네이션을 보여준다.
        - 안내 문구를 보여준다.
        - 안내 문구 : 모든 일정을 무사히 완료하면 포인트를 지급해 드려요!
        - 꺽새 아이콘을 눌러 다음 온보딩 UI로 넘어가거나 전 온보딩 UI로 넘어갈 수 있다.
    - 온보딩 step-3
        - 건너뛰기 버튼을 눌러 온보딩 UI를 끌 수 있다.
        - 페이지 네이션을 보여준다.
        - 안내 문구를 보여준다.
        - 안내 문구 : 포인트를 다 모았다면 상점에서 식물을 구매할 수 있어요! 포인트를 모아서 식물을 심어 주세요!
        - 꺽새 아이콘을 눌러 다음 온보딩 UI로 넘어가거나 전 온보딩 UI로 넘어갈 수 있다.

    - 요청이 성공 했다면 도메인 /login으로 이동 한다.

## 주문 관리

도메인 : /admin/orders

1. 헤더
    - 헤더 - 공통 UI 참조
2. 주문 개수
    - 들어온 주문이 몇 개인지 알 수 있다.
3. 주문 목록 테이블
    - 테이블 헤드 속성들
    - 속성 : 주문 고객, 주문 상품, 주문 번호, 주문 상태, 주소지
    - 테이블 항목
    - 주문 고객의 닉네임을 알 수 있다.
    - 주문 상품의 상품명을 알 수 있다.
    - 주문 번호의 8자리를 알 수 있다.
    - 주문 상태의 상태를 초록색, 빨간색으로 구분할 수 있다.
    - 상태 : 배송 중, 주문 접수 중, 주문 취소 요청
    - 주문자의 주소지를 알 수 있다.
    - 항목을 클릭하면 배송 모달, 주문 취소 요청 모달을 띄울 수 있다.
    - 주문 접수 중 상태의 항목을 클릭하면 배송 모달을 띄운다.
    - 주문 취소 요청 상태의 항목을 클릭하면 주문 취소 요청 모달을 띄운다.
    - 배송 모달 / 주문 취소 모달
    - 현재 주문 상태의 처리를 알 수 있다
    - 상품의 이미지를 볼 수 있다.
    - 주문 번호를 알 수 있다.
    - 주문 고객의 닉네임을 알 수 있다.
    - 주문 상품의 상품명을 알 수 있다.
    - 주문자의 주소지를 알 수 있다.
    - CTA 버튼을 눌러 주문을 처리할 수 있다.
    - 버튼 : 배송, 주문 취소
    - 배송 모달의 버튼을 눌러 주문 접수 중 상태를 배송 중으로 변경 한다.
    - 주문 취소 모달의 버튼을 눌러 취소 접수 중 상태를 주문 취소로 변경할 수 있다.
    - 주문 취소 요청이 성공하면 주문 리스트에서 해당 리스트를 삭제 한다.

## 상점 관리

도메인 : /admin/products

1. 헤더
    - 헤더 - 공통 UI 참조
2. 주문 개수
    - 등록 된 제품이 몇 개인지 알 수 있다.
3. 등록 된 제품의 목록을 볼 수 있다.
    - 항목에 제품의 이미지를 볼 수 있다.
    - 항목을 통해 제품의 카테고리를 알 수 있다.
    - 항목을 통해 제품의 제품명을 알 수 있다.
    - 항목을 통해 제품의 가격을 알 수 있다.
    - 수정 버튼을 눌러 제품을 수정할 수 있다.
    - 수정 버튼을 누르면 기본인 제품 추가 폼이 제품 수정 폼으로 변경 된다.
    - 삭제 버튼을 눌러 리스트에 제품을 삭제할 수 있다.
4. 제품 추가 폼
    - 사진 추가 버튼을 눌러 사진을 추가할 수 있다.
    - 제품명 입력창을 통해 제품명을 작성할 수 있다.
    - 제품 가격 입력창을 통해 제품 가격을 작성할 수 있다.
    - 제품 카테고리 드롭다운을 통해 제품의 카테고리를 정할 수 있다
    - 제품 카테고리 : 씨앗, 식물, 묘목
5. 제품 수정 폼
    - 모든 입력창에 기본적으로 수정 전의 데이터가 작성 돼 있다.
    - 사진 변경 버튼을 눌러 사진을 변경할 수 있다.
    - 제품명 입력창을 통해 제품명을 작성할 수 있다.
    - 제품 가격 입력창을 통해 제품 가격을 작성할 수 있다.
    - 제품 카테고리 드롭다운을 통해 제품의 카테고리를 정할 수 있다
    - 제품 카테고리 : 씨앗, 식물, 묘목
